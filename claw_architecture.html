High-Level Data Flow

sequenceDiagram
  participant User
  participant CLI
  participant Runtime as ConversationRuntime
  participant Session
  participant API as ProviderClient
  participant Tools as ToolExecutor
  participant Disk as .claw/sessions/

  User->>CLI: Input message
  CLI->>Runtime: run_turn(user_input)
  Runtime->>Session: push_user_text()
  Session->>Disk: Append JSONL record

  loop Until no tool uses or max iterations
    Runtime->>API: stream(ApiRequest)
    API-->>Runtime: AssistantEvent stream
    Runtime->>Session: push_message(assistant)

    opt Tool uses present
      loop For each tool use
        Runtime->>Tools: execute(tool_name, input)
        Tools-->>Runtime: Result or Error
        Runtime->>Session: push_message(tool_result)
      end
    end
  end

  Runtime->>Runtime: maybe_auto_compact()
  Runtime-->>CLI: TurnSummary
  CLI-->>User: Display response

Key Traits

  • ApiClient — stream model requests
  • ToolExecutor — dispatch tool calls
  • Provider — send/stream messages
  • PermissionPrompter — interactive auth
  • HookProgressReporter — hook events

Persistence

  • Sessions stored as JSONL in .claw/sessions/
  • Prompt cache in ~/.claude/cache/prompt-cache/
  • File rotation at 256KB (max 3 backups)
  • Atomic writes prevent corruption

Providers

  • Anthropic — Claude models
  • Xai — Grok models
  • OpenAi — OpenAI-compatible
  • Auto-detected from model name + env vars

Tool Count: 40+

  • File ops: bash, read, write, edit, glob, grep
  • Web: fetch, search
  • Agent/Task: spawn, manage, observe
  • Worker/Team/Cron management
  • LSP, MCP, Plugins

Persistence Format

  • JSONL (JSON Lines) in .claw/sessions/{id}.jsonl
  • Record types: session_meta, message, compaction
  • Append-only with atomic full-snapshot on rotation
  • Rotation threshold: 256KB, keeps max 3 backups

Session Control

  • create_managed_session_handle(id)
  • resolve_session_reference("latest")
  • Fork creates new session preserving messages
  • ManagedSessionSummary for metadata-only loads

Configuration

  • preserve_recent_messages: default 4
  • max_estimated_tokens: default 10,000
  • Auto-compact threshold: 100K input tokens
  • Env: CLAUDE_CODE_AUTO_COMPACT_INPUT_TOKENS

Summary Generation

  • Counts messages by role
  • Extracts unique tool names used
  • Collects recent user requests (max 3, 160 chars)
  • Wraps output in <summary> XML tags

Agent Spawning & Subagent System 
sequenceDiagram
  participant Parent as Parent Runtime
  participant Tools as Tools::run_agent()
  participant Thread as Spawned Thread
  participant SubRT as Subagent ConversationRuntime
  participant SubAPI as ProviderRuntimeClient
  participant SubTool as SubagentToolExecutor

  Parent->>Tools: execute("Agent", input)
  Tools->>Tools: Generate agent_id, resolve model
  Tools->>Tools: Determine allowed_tools by subagent_type
  Tools->>Thread: spawn_agent_job()

  Thread->>SubRT: build_agent_runtime()
  SubRT->>SubAPI: stream(request with filtered tools)
  SubAPI-->>SubRT: AssistantEvent stream

  loop Tool calls
    SubRT->>SubTool: execute(tool_name, input)
    SubTool->>SubTool: Check tool in allowed_tools
    SubTool->>Tools: execute_tool_with_enforcer()
    Tools-->>SubTool: Result
    SubTool-->>SubRT: Result
  end

  Thread->>Thread: Persist output + status
  Thread-->>Parent: AgentOutput manifest

Subagent Types & Tool Access

  • Explore: read, glob, grep, web, search (read-only)
  • Plan: read, glob, grep, web, todo, structured output
  • Verification: bash + read + glob + grep + web + todo
  • Default (generic): all tools except Team/Worker/MCP advanced

Agent Lifecycle

  • Thread named clawd-agent-{id}
  • Max 32 iterations per turn
  • Panic handler wraps execution
  • Status: running → completed | failed

Provider Configuration

Anthropic

  • Auth: ANTHROPIC_API_KEY or OAuth
  • Base URL: https://api.anthropic.com
  • Retry: 2 retries, exponential backoff 200ms-2s
  • Features: prompt cache, OAuth refresh

xAI (Grok)

  • Auth: XAI_API_KEY
  • Base URL: https://api.x.ai/v1
  • OpenAI-compatible protocol
  • Response normalization layer

OpenAI

  • Auth: OPENAI_API_KEY
  • Base URL: https://api.openai.com/v1
  • OpenAI-compatible protocol
  • Response normalization layer

Auth Sources

  • None — no auth
  • ApiKey(String) — x-api-key header
  • BearerToken(String) — Bearer header
  • ApiKeyAndBearer — both headers

TTL Defaults

  • Completion TTL: 30 seconds
  • Prompt TTL: 5 minutes
  • Cache break min drop: 2,000 tokens

Stats Tracked

  • Completion cache hits / misses / writes
  • Expected / unexpected invalidations
  • Total cache creation + read tokens
Error Handling & Retries

ApiError Variants

  • MissingCredentials
  • ExpiredOAuthToken
  • Http(reqwest::Error)
  • Api { status, error_type, message, retryable }
  • RetriesExhausted
  • InvalidSseFrame

Retry Policy

  • Retryable: 408, 409, 429, 500, 502, 503, 504
  • Max retries: 2
  • Backoff: 200ms → 400ms → 800ms (cap 2s)
  • Timeout/connect errors are retryable
